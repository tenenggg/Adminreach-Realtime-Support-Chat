ADMINREACH: COMPLETE DEPLOYMENT GUIDE
====================================

Follow these steps to set up the system on your company server.




1. PREPARE THE SERVER (The Master Computer)
-------------------------------------------

A. BASE SOFTWARE INSTALLATION:
   1. Download and Install PostgreSQL:
      - Go to: https://www.postgresql.org/download/windows/
      - During install, it will ask for a "Password". Choose something you'll remember (example: 1234).
      - After install, open "pgAdmin 4" (this is the database manager that comes with Postgres).
      - QUICK TEST: Open Command Prompt and type: psql -U postgres
        - If it asks for a password, your installation is successful.
        - If it says "not recognized", PATH is not set up. (Optional fix: search for "Edit the system environment variables" in Windows Start, click Environment Variables, edit Path, and add C:\Program Files\PostgreSQL\16\bin).
        - Note: If you do not want to fix terminal PATH, you can use pgAdmin only.

   2. Node.js (Optional but Recommended):
      - Go to: https://nodejs.org/ (Download the LTS version).
      - Note: Your backend .exe can run without system Node.js, but Node.js is useful for troubleshooting and PM2 setup.

B. DATABASE SETUP:
   1. Open pgAdmin 4.
   2. On the left, right-click Databases -> Create -> Database. Name it: adminreach
   3. Right-click your new adminreach database -> Select Query Tool.
   4. Open the setup.sql file from this project: database\setup.sql
   5. Copy ALL text from setup.sql.
   6. Paste into the query editor and press Execute Query (or F5).
   7. Your database tables are now ready.

C. FILE DEPLOYMENT:
   1. Create a folder named C:\AdminReach on the server.
   2. Copy the entire Backend folder from this project into C:\AdminReach.
      - Final location should be: C:\AdminReach\Backend
      - This supports BOTH deployment choices:
        a) Running server executable
        b) Running server.js with PM2

   6. CUSTOMIZE YOUR SERVER .ENV SETTINGS:
      Open C:\AdminReach\Backend\.env in Notepad and update values for your server:
      - PORT=5000
      - DB_USER=postgres
      - DB_PASSWORD=XXXX (your PostgreSQL password)
      - DB_HOST=localhost
      - DB_PORT=5432
      - DB_NAME=adminreach (change if your database name is different)
      - JWT_SECRET=change_this_to_a_strong_secret
      - IP_ADDRESS=YOUR_SERVER_IPV4 (example: 169.0.0.69)

   7. Start the backend using ONE of these:
      - Option A (Executable): double-click your server executable inside C:\AdminReach\Backend
      - Option B (PM2): use PM2 steps in Section 3 from C:\AdminReach\Backend
      - If you see a PostgreSQL connected message or server running log, the server is live.

D. FINAL FOLDER CHECKLIST (Very Important):
   Before starting the server, your folder should look like this:

   C:\AdminReach\
     |- Backend\
        |- adminreachServer.exe   (or your backend exe name, if using executable mode)
        |- server.js              (if using PM2 mode)
        |- package.json
        |- .env
        |- frontend\
           |- index.html
           |- css\
           |- js\
           |- login.html
           |- register.html
           |- chat.html
           |- admin.html

E. FIREWALL (CRITICAL):
   - You must allow your backend port (default: 5000) in Windows Firewall on the server.
   - If this is blocked, client computers cannot connect.




2. PREPARE THE CLIENTS (Employee Computers)
--------------------------------------------

1. Copy the client win-unpacked folder to each employee computer.
2. Inside that folder, open .env with Notepad.
3. Update the server URL:
   - Example: SERVER_URL=http://192.168.1.105:5000
   - Tip: On the Master Computer, run ipconfig in Command Prompt and use the IPv4 Address.
4. Save and close .env.
5. Double-click AdminReach.exe to run the app.
6. Optional: Right-click AdminReach.exe -> Send to -> Desktop (create shortcut).




3. ADVANCED: AUTOMATING THE SERVER (PM2)
-----------------------------------------
Use this if you run backend as Node.js script and want auto-restart and boot persistence.

1. INSTALL TOOLS (Run once):
   npm install pm2 -g
   npm install pm2-windows-startup -g
   pm2-startup install

2. START THE SERVER (Go into Backend folder first):
   cd C:\AdminReach\Backend
   pm2 start server.js --name "adminreach"
   pm2 save (IMPORTANT, IF NOT IT WOULDNT BE THE SAME AFTER RESTARTING SERVER OR PC)

3. USEFUL COMMANDS:
   - pm2 status       (Check online/offline state)
   - pm2 logs         (View runtime logs)
   - pm2 restart all  (Apply config or env changes)
   - pm2 stop all     (Stop all managed processes)



NOTES
----- 
- Project backend folder in this repo is named: Backend
- SQL file location in this repo is: database\setup.sql
- Client executable in current build is: AdminReach.exe
- Client env key is: SERVER_URL
